# CLAUDE.md — project guide for Claude Code

This file is auto-loaded every session. It orients you (and any contributor) to what
this repo is, how to run each piece, and the conventions to follow. If you're a
returning Claude session, also read your **memory** folder (see "Continuity" below) —
it carries the project history and decisions that don't belong in a public repo.

## What this project is

**mnm-tools** — a set of companion tools for the MMO *Monsters & Memories*. Four
subsystems live in this one repo:

| Subsystem | Lives in | What it is | Run / deploy |
|-----------|----------|-----------|--------------|
| **MnM Map** (desktop app) | `main.js`, `preload.js`, `renderer/` | Electron overlay: zone maps + markers companion | `npm start` · build `npm run dist` · release `npm run release` |
| **MnMdb** (website) | `mnmdb/` | Static SPA at **mnm-db.com** — item/mob/zone DB, drop rates, auction prices, maps. No build step. | `npx serve mnmdb -l 5601` → http://localhost:5601 · deploys via **GitHub Pages** (`mnmdb/CNAME`) |
| **API** (backend) | `api/` | Cloudflare **Worker + D1 + R2** — community map/marker submissions, Discord login, admin/moderation | `cd api && npx wrangler deploy` (see `api/README.md`, `api/schema.sql`) |
| **Auction tracker** | `tracker/` | OCRs the LiveMNM stream → per-server auction prices; parses game Ledger files | Runs 24/7 on the **VPS** (see `VPS-SETUP.md`), not locally |

## Running locally

- **Website:** `npx serve mnmdb -l 5601`, open http://localhost:5601. It's plain
  HTML/CSS/JS — edit `mnmdb/app.js` / `mnmdb/styles.css` and reload. Data comes from
  the `*.json` files in `mnmdb/` (all generated/published; don't hand-edit).
- **Desktop app:** `npm install` once, then `npm start`.
- **API:** `cd api && npx wrangler dev` (needs `wrangler login`).

## Key files

- `mnmdb/app.js` — the entire website SPA. Hash routing in `route()`; each page is a
  `renderX()` function writing to `#content`. Data loads in `loadWikiStats()` and the
  init block at the bottom.
- `mnmdb/wiki.json`, `data.json`, `auctions.json`, `vendors.json`, … — published data
  the site reads. Regenerated by `tracker/` scripts; the VPS commits `auctions.json`.
- `tracker/parse-auctions.js`, `publish-auctions.js`, `capture-auctions.js` — the
  auction pipeline. `tracker/ledger-parser.js` decodes the game's local Ledger logs.
- `api/src/index.js` — the Worker. `api/wrangler.toml` — its config/bindings.

## Conventions

- **Pull before you work.** The VPS auto-commits fresh `auctions.json` (and enriched
  `wiki.json`) to `main` throughout the day. Always `git pull --rebase` first, or your
  push will be rejected as "behind."
- **Commit only when asked**, and never bypass the pre-commit secret scanner
  (`.githooks/pre-commit`) with `--no-verify`. If it flags a genuinely-public token,
  add it to `.githooks/allowed-secrets` instead. (The scanner is only active if
  `git config core.hooksPath .githooks` has been run in this clone — it's machine-local
  and not carried by `git clone`; see `SETUP.md`.)
- **Match the surrounding code.** `mnmdb/app.js` is terse vanilla JS with dense inline
  comments explaining *why*; keep that style. No framework, no build tooling.
- **Data files are generated** — fix the generator in `tracker/`, don't hand-edit the
  JSON in `mnmdb/`.
- **This repo is public.** Don't commit secrets, personal data, or private context.
